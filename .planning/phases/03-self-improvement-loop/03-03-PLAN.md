<objective>
Wire the POST /confirm endpoint that triggers the complete rule generation pipeline: receive operator confirmation, spawn background task, stream Opus 4.6 rule generation via WebSocket, validate, write .py file + Aerospike provenance, hot-reload SafetyGate. Handles both new generation and evolution paths.

Purpose: This is the API surface for the self-improvement loop. Judges click "confirm attack" and watch Python code stream onto the dashboard in real time. The route connects RuleGenerator (Plan 01) and rule_store (Plan 02) into a live pipeline.

Output: `sentinel/api/routes/confirm.py` with full generation/evolution pipeline, registered in FastAPI app, tested.

    **Step 1: Create `sentinel/api/routes/confirm.py`:**

    Request/response models:
    ```python
    class ConfirmRequest(BaseModel):
        episode_id: str
        attack_type: str  # e.g. "prompt_injection", "identity_spoofing"

    class ConfirmResponse(BaseModel):
        episode_id: str
        status: str  # "accepted"
    ```

    Route handler per D-01, D-02:
    ```python
    @router.post("/confirm", status_code=202, response_model=ConfirmResponse)
    async def confirm(req: ConfirmRequest) -> ConfirmResponse:
        from sentinel.api.main import app_state
        episode = app_state["active_episodes"].get(req.episode_id)
        if episode is None:
            raise HTTPException(status_code=404, detail=f"Episode {req.episode_id} not found in active cache")
        asyncio.create_task(_run_rule_pipeline(req, episode, app_state))
        return ConfirmResponse(episode_id=req.episode_id, status="accepted")
    ```

    Background pipeline `async def _run_rule_pipeline(req, episode, app_state)`:

    1. Extract VerdictBoard and prediction errors from episode:
       - `vb_dict = episode.verdict_board.model_dump()` if Episode object, or `episode["verdict_board"]` if dict
       - `prediction_errors = episode.prediction_report or {}`
       - Handle both Episode model instances and raw dicts (active_episodes may store either)

    2. Update episode with operator confirmation:
       - Set `episode.operator_confirmation = "confirmed_attack"`
       - Set `episode.attack_type = req.attack_type`

    3. Detect generation vs evolution (per D-08):
       - If `episode.generated_rules_fired` is non-empty → evolution path
       - Else → new generation path

    4. **New generation path:**
       - Create `RuleGenerator(app_state["llm_client"], app_state["models"]["supervisor"])`
       - Define `ws_broadcast` wrapper: `async def _broadcast(event, episode_id, data): await ws_manager.broadcast(event, episode_id, data)`
       - Call `source, error = await generator.generate(req.attack_type, vb_dict, prediction_errors, ws_broadcast=_broadcast, episode_id=req.episode_id)`
       - If source is None: broadcast `rule_generation_failed` (already done in generate()), return
       - Get `rule_id = await next_rule_id(app_state["aerospike"])` if aerospike available, else `"rule_001"`
       - Write .py file to `sentinel/gate/rules/rule_generated_{NNN}.py` where NNN is from rule_id (e.g. "001" from "rule_001"). Use `Path(__file__).parent.parent.parent / "gate" / "rules" / f"rule_generated_{rule_id.split('_')[-1]}.py"`. File content = the source string.
       - Call `app_state["safety_gate"].load_rules_from_directory(rules_dir)` for hot reload (per D-03)
       - Write provenance to Aerospike: `write_latency = await write_rule(rule_id, source, [req.episode_id], prediction_errors, version=1, client=app_state["aerospike"])`
       - Update episode: `episode.generated_rule_source = source`, `episode.new_rules_deployed = [rule_id]`
       - Broadcast `rule_deployed` event with data: `{"rule_id": rule_id, "version": 1, "source": source, "episode_ids": [req.episode_id], "write_latency_ms": write_latency, "attribution": f"Generated Rule #{rule_id.split('_')[-1]} (learned from Episode #{req.episode_id[:8]}) | Deployed 0s ago"}`

    5. **Evolution path (per D-08, D-09, D-10):**
       - Get the existing rule_id from `episode.generated_rules_fired[0]`
       - Load existing rule source from Aerospike: `existing = await app_state["aerospike"].get("rules", existing_rule_id)` and extract `source`, `episode_ids`
       - Find original episode VB: read original episode from active_episodes or Aerospike via `get_episode(original_episode_id)`
       - Get original prediction errors from the original episode's prediction_report
       - Call `source, error = await generator.evolve(existing_source, req.attack_type, original_vb, current_vb, original_pe, current_pe, ws_broadcast=_broadcast, episode_id=req.episode_id)`
       - If source is None: broadcast failure, return
       - Overwrite the .py file on disk (same filename as v1) per D-10
       - Call `load_rules_from_directory()` for hot reload
       - Write to Aerospike with `version=existing_version+1`, `episode_ids=[...original_ids, req.episode_id]`, combined prediction errors
       - Broadcast `rule_deployed` with version=2

    6. Wrap entire pipeline in try/except — any uncaught exception broadcasts `rule_generation_failed` to prevent silent failures.

    **Step 2: Register the router in `sentinel/api/main.py`:**
    - Add `from sentinel.api.routes.confirm import router as confirm_router` alongside the existing investigate_router import
    - Add `app.include_router(confirm_router)` alongside the existing `app.include_router(investigate_router)`

    **Step 3: Update `sentinel/api/routes/__init__.py`** if needed to export the new router.
