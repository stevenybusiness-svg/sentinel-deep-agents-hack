    ```python
    """Rule store — MEM-02, MEM-05.
    Reads and writes generated scoring rules to the Aerospike sentinel.rules set.
    write_rule() measures and returns write latency in ms for dashboard display.
    """
    import json
    import time
    from sentinel.memory.aerospike_client import AerospikeClient

    RULES_SET = "rules"
    ```

    **Functions to implement:**

    1. `async def write_rule(rule_id: str, source: str, episode_ids: list[str], prediction_errors: dict, version: int, client: AerospikeClient) -> float`:
       - Measures latency with `time.perf_counter()` (same as episode_store)
       - Bins per D-04: `rule_id` (str), `source` (str), `episode_ids` (json.dumps list), `prediction_errors` (json.dumps dict), `timestamp` (int ms via `int(time.time() * 1000)`), `version` (int), `fire_count` (0 for new rules)
       - Calls `client.put(RULES_SET, rule_id, bins)`
       - Calls `_update_rules_index(rule_id, client)` after write
       - Returns `round(latency_ms, 2)`

    2. `async def _update_rules_index(rule_id: str, client: AerospikeClient) -> None`:
       - Read `__rules_index__` key from RULES_SET (try/except, default to empty list)
       - Parse JSON list of rule IDs
       - Append rule_id if not already present
       - Write back as JSON under `__rules_index__` key with bin `{"rule_ids": json.dumps(updated_list)}`

    3. `async def load_all_rules(client: AerospikeClient) -> list[dict]`:
       - Read `__rules_index__` key from RULES_SET
       - Parse JSON list of rule IDs
       - For each rule_id, `client.get(RULES_SET, rule_id)` and deserialize JSON bins (episode_ids, prediction_errors)
       - Return list of rule dicts (each with rule_id, source, episode_ids as list, prediction_errors as dict, version, fire_count, timestamp)
       - On any error, return empty list (graceful degradation)

    4. `async def increment_fire_count(rule_id: str, client: AerospikeClient) -> None`:
       - Read current bins via `client.get(RULES_SET, rule_id)`
       - Increment `fire_count` by 1
       - Write back via `client.put(RULES_SET, rule_id, bins)`
       - Wrap in try/except — fire_count is telemetry, never block on failure

    5. `async def next_rule_id(client: AerospikeClient) -> str`:
       - Read `__rule_counter__` key from RULES_SET (try/except, default count=0)
       - Increment count by 1
       - Write back `{"count": n}` to `__rule_counter__`
       - Return `f"rule_{n:03d}"` (e.g. "rule_001", "rule_002")
