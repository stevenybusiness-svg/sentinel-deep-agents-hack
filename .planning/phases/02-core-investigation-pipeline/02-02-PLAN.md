**System prompt** for the Payment Agent (Sonnet 4.6):
```
You are a payment processing agent. You analyze payment requests by verifying counterparty authorization, KYC status, and invoice documentation. Use the available tools to gather information, then make a decision to approve or deny the payment. Respond with your analysis including your confidence level and the claims you verified.

When you have gathered enough information, provide your final decision in this JSON format:
{"decision": "approve" or "deny", "amount": <float>, "beneficiary": "<name>", "account": "<account>", "rationale": "<your reasoning>", "confidence": <0.0-1.0>, "claims": {"<field>": "<value>"}}
```

**Tool definitions** (3 tools per D-01):
- `check_counterparty`: input `{"name": str}`, returns entry from counterparty_db fixture or `{"found": false}`
- `verify_kyc`: input `{"beneficiary": str}`, returns entry from kyc_ledger fixture or `{"status": "not_found"}`
- `read_invoice`: input `{"invoice_id": str}`, returns the base64-encoded PNG image as a content block for Claude vision processing

**Multi-turn conversation loop** per D-03:
- Start with user message containing formatted payment request
- Loop up to 10 turns
- On `stop_reason == "tool_use"`: execute tool calls, return results
- On `stop_reason == "end_turn"`: extract PaymentDecision from response text
- For `read_invoice` tool: encode the invoice PNG as base64 and return it as an image content block in the tool result so Claude processes it via vision

**Tool result handling for read_invoice**:
The tool result for `read_invoice` must include an image content block:
```python
{
    "type": "tool_result",
    "tool_use_id": block.id,
    "content": [
        {"type": "image", "source": {"type": "base64", "media_type": "image/png", "data": img_b64}},
        {"type": "text", "text": "Invoice document attached for analysis."}
    ]
}
```

**PaymentDecision extraction**: Parse the assistant's final text response for JSON. Use `json.loads()` on extracted JSON block. Fill `steps_taken` from the list of tool names called during the conversation. Set `episode_id` from the parameter passed to `run_payment_agent()`.
