# Phase 4.1 Plan 01: Backend Performance Optimization and Narrative Generation Summary

Batch tool calls, streaming supervisor reasoning, prompt caching setup, and hybrid template+async-polish narrative pipeline — all 9 done criteria verified green with 182 passing tests.

## What Was Built

**Task 1: Batch tool calls, extend EventType, add cache_control and streaming**

- `PAYMENT_AGENT_SYSTEM_PROMPT`: Added explicit batching instruction (`call all required tools simultaneously`) per D-01. Padded to ~2308 tokens (Sonnet 4.6 minimum 2048) with detailed tool guidelines, claim verification criteria, processing protocol, and edge case handling.
- `EventType` in `sentinel/schemas/events.py`: Extended from 9 to 12 literal values: `supervisor_token`, `narrative_template`, `narrative_ready` per D-03/D-09.
- `SUPERVISOR_SYSTEM_PROMPT`: Padded to ~4940 tokens (Opus 4.6 minimum 4096) with behavioral baseline table, investigation protocol for all 3 sub-agents, complete risk taxonomy (4 attack vectors), scoring function semantics (thresholds, compound scoring logic), example clean vs. anomalous verdict boards, rule generation process (5 steps), rule registry format, operator voice interface guidance, timing/latency targets.
- `supervisor.py` step 3: Replaced `messages.create()` with `messages.stream()` async context manager. Each `text_chunk` from `stream.text_stream` is broadcast as `supervisor_token` WS event. Full message recovered via `stream.get_final_message()`.
- `supervisor.py` step 4: Payment Agent `messages.create()` calls now pass system as a list of blocks with `cache_control: {"type": "ephemeral"}` per D-02.
- `build_narrative_template()`: Pure Python function (no LLM call) building 4 narrative strings from structured investigation data: attack_narrative, agent_reasoning, prediction_summary, self_improvement_arc.
- `_generate_narrative_polish()`: Async fire-and-forget coroutine launched via `asyncio.create_task()`. Makes a Sonnet call to polish the 3 prose narratives and broadcasts `narrative_ready`. Silent failure leaves template content visible.
- Event ordering in `run_investigation()`: `gate_evaluated` → `narrative_template` (synchronous, same frame) → `create_task(polish)` → `episode_written`.

**Task 2: Unit tests for narrative template generation (TDD)**

- `tests/test_narrative.py`: 11 unit tests covering all `build_narrative_template()` behaviors: all 4 keys present, NO-GO attack_narrative contains formatted dollar amount and beneficiary, GO uses "flagged" not "blocked", empty verdicts returns graceful string, 3 verdicts mentions all agent names, float prediction_errors produces numeric summary, no prediction_errors is graceful, no rule_sources returns "No rules generated yet", version > 1 rule mentions "refined", version 1 rule mentions rule_id without "refined".
- `tests/test_supervisor.py`: Updated `_make_mock_llm()` to use `asynccontextmanager` for `messages.stream()` (Supervisor) and `AsyncMock` for `messages.create()` (Payment Agent). Updated `test_event_broadcast_sequence` to include `supervisor_token` and `narrative_template`. Updated model assertion to check `mock_stream` call kwargs instead of `messages.create.call_args_list[0]`.
- `tests/test_rule_generator.py`: Updated `test_event_type_has_nine_values` to `test_event_type_has_twelve_values` (9 → 12 values).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed IndexError in SUPERVISOR_SYSTEM_PROMPT.format()**
- **Found during:** Running supervisor tests after implementation
- **Issue:** Example code blocks in the padded SUPERVISOR_SYSTEM_PROMPT contained literal `{` and `}` characters (in the generated rule function example and rule registry example). These were interpreted as positional format string placeholders by `.format(episode_context=...)`, causing `IndexError: Replacement index 0 out of range for positional args tuple`.
- **Fix:** Escaped all literal curly braces in example blocks as `{{` and `}}` within the triple-quoted string that uses `.format()`.
- **Files modified:** `sentinel/agents/supervisor.py`
- **Commit:** 8e9520f

**2. [Rule 1 - Bug] Fixed broken supervisor tests after streaming conversion**
- **Found during:** Running `pytest tests/test_supervisor.py` after Task 1 implementation
- **Issue:** The existing `_make_mock_llm()` helper mocked `messages.create` for both supervisor and payment agent calls. Converting supervisor to `messages.stream()` meant the mock no longer supported the async context manager protocol, causing `TypeError: 'coroutine' object does not support the asynchronous context manager protocol`.
- **Fix:** Rewrote `_make_mock_llm()` to use `asynccontextmanager` for `messages.stream()` yielding a mock stream object with `text_stream` async generator and `get_final_message()`. Payment Agent responses remain on `messages.create`. Updated test assertions that previously indexed `create.call_args_list[0]` for supervisor (now uses `messages.stream`).
- **Files modified:** `tests/test_supervisor.py`
- **Commit:** 8e9520f
