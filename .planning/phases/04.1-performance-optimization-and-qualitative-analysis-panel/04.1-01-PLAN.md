<objective>
Optimize the backend investigation pipeline for demo-viable latency and add narrative generation.

Purpose: The investigation pipeline currently takes ~30s due to sequential Payment Agent tool calls. This plan reduces that to <10s via batched tool calls, adds prompt caching for repeat runs, streams Supervisor reasoning to the frontend to hide remaining latency, and adds the hybrid template-fill + async LLM polish narrative generation pipeline (backend side only -- frontend in Plan 02).

Output: Modified supervisor.py with streaming + caching + narrative generation, modified payment_agent.py with batched tool call prompt, extended EventType schema, unit tests for narrative template function.
</objective>

**1d. Convert Supervisor call to streaming (per D-03):**
In `supervisor.py` step 3, replace `llm_client.messages.create()` with `llm_client.messages.stream()` context manager. Change the system parameter from a plain string to a list of text blocks with `cache_control: {"type": "ephemeral"}` on the last block:

```python
supervisor_reasoning = ""
async with llm_client.messages.stream(
    model=models["supervisor"],
    max_tokens=1024,
    system=[{
        "type": "text",
        "text": supervisor_prompt,
        "cache_control": {"type": "ephemeral"},
    }],
    messages=[{...}],
) as stream:
    async for text_chunk in stream.text_stream:
        supervisor_reasoning += text_chunk
        await ws.broadcast("supervisor_token", episode_id, {"token": text_chunk})
    supervisor_response = await stream.get_final_message()  # recover full Message for usage stats / stop_reason
```

Also add `cache_control` to the Payment Agent system prompt in step 4 by passing system as a list of blocks:
```python
system=[{
    "type": "text",
    "text": PAYMENT_AGENT_SYSTEM_PROMPT,
    "cache_control": {"type": "ephemeral"},
}],
```

Do NOT apply streaming to Payment Agent calls -- only to the Supervisor reasoning call. Payment Agent calls remain as `messages.create()` because streaming breaks tool_use JSON parsing.

**1e. Add narrative template generation and async polish (per D-09):**
Add a new function `build_narrative_template()` to `supervisor.py` (or a new file `sentinel/narrative.py` -- use your judgment on file placement). This is a pure Python function (no LLM call) that builds plain English narrative strings from structured investigation data. Follow the template logic from RESEARCH.md Pattern 3 code example exactly.

Add a new async function `_generate_narrative_polish()` to `supervisor.py` that:
1. Takes episode_id, verdict_board, gate_result, verdicts, ws, llm_client, models as params
2. Builds a prompt asking Sonnet to rewrite 3 template narratives (attack_narrative, agent_reasoning, prediction_summary) in richer natural prose (2-4 sentences each, plain English, no technical jargon)
3. Calls `llm_client.messages.create(model=models["agent"], max_tokens=512, ...)`
4. Parses the response (use a simple JSON format or `---SECTION---` delimiter)
5. Broadcasts `narrative_ready` with the polished strings
6. Wraps everything in try/except -- silent failure leaves template content visible

1. Call `build_narrative_template()` synchronously to get template strings
2. Get active rule sources from safety_gate for self-improvement arc context
3. Broadcast `narrative_template` event with all 4 template strings
4. Fire `asyncio.create_task(_generate_narrative_polish(...))` -- fire-and-forget, MUST NOT await

The gate_evaluated broadcast and narrative_template broadcast should happen before the episode write (step 11). The polish task runs concurrently with the episode write.
