  <action>
    **Create vercel.json (per PHASE8-06):**

    1. Create `frontend/vercel.json`:
       ```json
       {
         "$schema": "https://openapi.vercel.sh/vercel.json",
         "buildCommand": "npm run build",
         "outputDirectory": "dist",
         "rewrites": [
           {
             "source": "/api/:path*",
             "destination": "https://YOUR_EC2_HOST/api/:path*"
           },
           {
             "source": "/(.*)",
             "destination": "/index.html"
           }
         ]
       }
       ```
       NOTE: The EC2 URL placeholder (`YOUR_EC2_HOST`) will be replaced in Phase 9 when the actual EC2 instance is provisioned. The API rewrite MUST come before the SPA catch-all (Pitfall 2 from research).

    **Parameterize WebSocket URL (per PHASE8-06):**

    2. In `frontend/src/hooks/useWebSocket.js`, replace the hardcoded WebSocket URL:
       - Find: `const ws = new WebSocket(`ws://${window.location.host}/ws`)`
       - Replace with:
         ```javascript
         const wsBase = import.meta.env.VITE_WS_URL || `ws://${window.location.host}`
         const ws = new WebSocket(`${wsBase}/ws`)
         ```
       This allows:
       - Local dev: falls back to `ws://localhost:5173` (Vite proxy handles it)
       - Vercel prod: uses `wss://EC2_HOST` set in Vercel environment variables

    **Parameterize API URL (per PHASE8-06):**

    3. In `frontend/src/App.jsx`, update the `runAttack` function:
       - Find: `await fetch('/api/investigate', {`
       - Replace with:
         ```javascript
         const apiBase = import.meta.env.VITE_API_URL || ''
         await fetch(`${apiBase}/api/investigate`, {
         ```
       This allows:
       - Local dev: empty string = same-origin (relative path)
       - Vercel with rewrites: empty string (rewrites handle routing to EC2)
       - Direct EC2: set VITE_API_URL to the EC2 URL

    4. Search for any other hardcoded API calls in the frontend (e.g., `/api/confirm`) and parameterize them too:
       - Use grep to find all `fetch('/api/` or `fetch("/api/` patterns
       - Each should use `${apiBase}/api/...` pattern
       - Define `apiBase` as a module-level constant or pass via context

    **Update .env.example:**

    5. In `.env.example`, add the Vercel-specific env vars if not already present:
       ```
       # Vercel deployment (set in Vercel dashboard, not .env)
       VITE_API_URL=
       VITE_WS_URL=
       ```
