**Image fix (DEMO-POLISH-01):** In `sentinel/api/main.py`, add two explicit FileResponse routes for `/invoice_clean.png` and `/invoice_forensic.png` BEFORE the SPA catch-all route. The root cause is that the catch-all `@app.get("/{full_path:path}")` intercepts PNG requests and returns index.html instead of the image files.

Add these two route handlers inside the `if _FRONTEND_DIST.exists():` block, between the `app.mount("/assets", ...)` line and the `@app.get("/{full_path:path}")` catch-all:

```python
@app.get("/invoice_clean.png", include_in_schema=False)
async def serve_invoice_clean() -> FileResponse:
    return FileResponse(str(_FRONTEND_DIST / "invoice_clean.png"), media_type="image/png")

@app.get("/invoice_forensic.png", include_in_schema=False)
async def serve_invoice_forensic() -> FileResponse:
    return FileResponse(str(_FRONTEND_DIST / "invoice_forensic.png"), media_type="image/png")
```

These MUST appear before the `@app.get("/{full_path:path}")` catch-all or FastAPI will still match the catch-all first.
