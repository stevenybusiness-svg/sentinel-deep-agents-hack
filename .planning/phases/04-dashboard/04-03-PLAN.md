```jsx
  </td>
  <td className="py-1.5 px-2">{renderSeverityBadge(claim.severity)}</td>
</tr>
```

**Severity badges (per UI-SPEC):**
```javascript
function renderSeverityBadge(severity) {
  const styles = {
    critical: 'bg-danger/20 text-danger border border-danger/30',
    warning: 'bg-warning/20 text-warning border border-warning/30',
    info: 'bg-primary/20 text-primary border border-primary/30',
  }
  return (
    <span className={`text-[11px] uppercase rounded-full px-1.5 py-0.5 ${styles[severity] || styles.info}`}>
      {severity}
    </span>
  )
}
```

**Prediction vs. actual sub-rows (DASH-10):**
For each mismatched field (where `claim.match === false`), check if `verdictBoard.prediction_errors` exists. If so, render a sub-row below the mismatch row:
```jsx
{!claim.match && predictionErrors && (
  <tr className="border-b border-border-muted/30">
    <td colSpan={5} className="py-1 px-2 pl-6 text-[11px] text-text-muted italic">
      Expected: {findExpectedValue(claim.field, predictionErrors)}
    </td>
  </tr>
)}
```

`findExpectedValue` looks into `predictionErrors.investigation_outcome_errors` or `predictionErrors.deviation_details` for the relevant field's predicted value. If not found, show the prediction summary: `z-score: {predicted_z_score}, summary: {summary_score}`.
