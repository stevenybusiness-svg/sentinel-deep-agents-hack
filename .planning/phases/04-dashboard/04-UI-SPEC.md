## Layout Contract

This is a fixed viewport dashboard — no scrolling during demo. All panels are visible simultaneously.

### Two-Column Grid

```
┌────────────────────────────────────────────────────────┐
│  header  h-12  (header bar — full width)               │
├───────────────────────┬────────────────────────────────┤
│                       │  Gate Decision                 │
│  Investigation Tree   │  Anomaly Score Bar             │
│  (@xyflow/react)      │  Verdict Board Table           │
│  full left-column     │  Forensic Scan (side-by-side)  │
│  height               │  Rule Source Panel             │
│                       │  Aerospike Latency             │
└───────────────────────┴────────────────────────────────┘
```

- Left column: `w-1/2` or `flex-1`, full `h-full`, no internal splits
- Right column: `w-1/2` or fixed width, scrollable internally (`overflow-y-auto`), panels stacked top-to-bottom
- Header: `h-12 px-4 border-b border-border-muted bg-surface/50 backdrop-blur-md sticky top-0 z-10` (existing pattern)

**Source:** CONTEXT.md D-01, D-02, D-03

### Header Bar Layout

```
Sentinel Dashboard    [Run Attack 1 — Invoice Injection] [Run Attack 2 — Identity Spoofing]   ● connected · idle
```

- Title: left-aligned `text-sm font-semibold tracking-tight text-white` (existing)
- Attack buttons: centered or right of title, disabled while `investigationStatus === 'running'`
- Status chip: right-aligned, existing `font-mono uppercase tracking-wider` with `pulse-dot`

**Source:** CONTEXT.md D-04, D-05, §Specifics header mockup

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, tight inline spacing |
| sm | 8px | Panel internal padding compact rows, label-to-value gaps |
| md | 16px | Default panel padding, element spacing |
| lg | 24px | Section padding within right column panels |
| xl | 32px | Layout column gap, major section spacing |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Header height: 48px (h-12 = 3rem = 48px) — standard Tailwind h-12
- Touch targets for attack buttons: minimum 40px height (h-10 = 2.5rem)
- Pulse dot size: 6px (w-1.5 h-1.5) — existing pattern, do not change

## Component Inventory

### Investigation Tree (DASH-01, DASH-02)

Node states and visual encoding:

| State | Ring / Border | Fill | Icon |
|-------|---------------|------|------|
| pending | `border-border-muted` (1px) | `surface` | Material: `hourglass_empty` (text-muted) |
| active | `border-accent` (2px) with pulse animation | `surface` | Material: `sync` (text-accent, spinning) |
| complete | `border-success` (1px) | `surface` | Material: `check_circle` (text-success) |
| blocked | `border-danger` (2px) | `rgba(248,81,73,0.1)` | Material: `block` (text-danger) |
| rule_node | `border-warning` (1px) | `rgba(227,179,65,0.1)` | Material: `auto_awesome` (text-warning) |

Node labels: 11px Inter 400, `text-text-main`.

Edge animation: `animated: true` on `@xyflow/react` edge config; color `#30363d` default, `#57abff` when data is flowing.

Tree structure:
```
[Supervisor] → [Payment Agent]
             → [Risk Agent]
             → [Compliance Agent]
             → [Forensics Agent]
             → [Safety Gate]
             → [Rule: Generated #001]   (appears after rule_deployed event)
             → [Rule: Generated #001 v2] (replaces or extends after evolution)
```

**Source:** CONTEXT.md D-01, DASH-01, DASH-02, CONTEXT.md §Claude's Discretion

### Gate Decision Panel (DASH-07)

```
┌─────────────────────────────────────────────────────┐
│  ⛔ NO-GO                           Score: 1.84      │
│  Attribution: rule_mismatch, generated_rule_001 ...  │
│                                                      │
│  [ Confirm Attack — Learn ▶ ]                        │
└─────────────────────────────────────────────────────┘
```

- Gate verdict text: Display role — 20px Inter 600. Color: `text-danger` (NO-GO), `text-success` (GO), `text-warning` (ESCALATE)
- Score line: 13px Inter 400 `text-text-muted`
- Attribution text: 12px Inter 400 `text-text-main`, wraps below score
- "Confirm Attack — Learn ▶" button: only visible on NO-GO; `bg-accent text-white font-semibold text-sm px-4 py-2 rounded`; changes to `"Generating rule..."` (disabled, text-muted) after click
- Blinking cursor in rule source panel while waiting for first token: `animate-pulse` on a `|` character

**Source:** CONTEXT.md D-06, D-07, D-08, §Specifics gate mockup

### Composite Anomaly Score Bar (DASH-11)

- Horizontal bar, full panel width, height 24px
- Segmented fill: each rule's contribution is a color-coded segment with width proportional to its score contribution
  - Hardcoded rule segments: `bg-primary` (`#57abff`)
  - Generated rule segments: `bg-danger` (`#f85149`)
- Threshold line: 2px vertical line at the threshold position (score = 1.0), color `var(--accent)` (`#aa3bff`)
- Score value label: right-aligned 12px Roboto Mono `text-text-main`
- Rule contribution labels: 10px below each segment on hover or always if space permits

**Source:** CONTEXT.md D-03 (position), DASH-11

### Verdict Board Table (DASH-03, DASH-10)

- Table: full panel width, compact rows (`py-1.5 px-3`)
- Columns: Field | Agent Claimed | Independently Found | Match | Severity
- Severity badges: Label role — 11px Inter 400 uppercase, pill shape `rounded-full px-1.5 py-0.5`
  - `critical`: `bg-danger/20 text-danger border border-danger/30`
  - `warning`: `bg-warning/20 text-warning border border-warning/30`
  - `info`: `bg-primary/20 text-primary border border-primary/30`
- Mismatch rows: background `rgba(248,81,73,0.05)` on critical rows
- Match indicator: `text-success` checkmark icon / `text-danger` x icon in Match column
- Prediction vs. actual: inline sub-row below each mismatched field showing `Expected: {value}` in `text-text-muted` italic 11px. This is the prediction vs. actual integration (DASH-10), surfaced within the table rather than as a separate panel.

**Source:** CONTEXT.md D-03, D-07 (integrated prediction panel), DASH-03, DASH-10

### Forensic Scan Panel (DASH-04)

- Two-up layout: left = "Invoice (Agent View)" / right = "Forensic Scan"
- Each pane: equal width, fixed height (~160px), `object-fit: contain`, `bg-surface rounded border border-border-muted`
- Label above each pane: Label role — 11px Inter 400 uppercase `text-text-muted`
- Hidden text annotation in forensic scan: red overlay highlight (`rgba(248,81,73,0.4)` box) drawn via absolute positioning over the detected region
- Panel is always rendered; shows placeholder "No documents attached" when forensics agent found no attachments

**Source:** CONTEXT.md D-03 (position), DASH-04

### Rule Source Panel (DASH-05)

```
def score(verdict_board: dict) -> float:  [v2]
    """Evolved: drops single-attack artifacts,
    strengthens compound behavioral signals."""
    ...

―― Provenance ――
Episode: ep_abc123 · Deployed: 30s ago
Prediction errors: confidence_z_score +2.4σ, unable_to_verify [beneficiary]
```

- Code block: `bg-[#1f2028] rounded border border-border-muted p-3 font-mono text-[12px] leading-relaxed overflow-y-auto`
- Syntax highlighting via CSS classes (no heavy library):
  - Keywords (`def`, `return`, `if`, `else`, `float`, `dict`): `text-primary` (`#57abff`)
  - Strings and docstrings: `text-success` (`#3fb950`)
  - Numbers: `text-warning` (`#e3b341`)
  - Comments: `text-text-muted` italic
  - Default: `text-text-main`
- `[v2]` badge: inline after function signature, `bg-warning/20 text-warning text-[10px] font-mono rounded px-1`
- Provenance section: `border-t border-border-muted mt-3 pt-2 text-[11px] text-text-muted font-mono`
- Horizontal rule label `―― Provenance ――`: `text-text-muted` centered or left-aligned
- Streaming: tokens appended character-by-character via `streamingBuffer`; panel auto-scrolls to bottom; after `rule_deployed` event, final source replaces streamed content

**Source:** CONTEXT.md D-09, D-10, D-11, D-12, §Specifics rule panel mockup

### Trust Score Bar (DASH-06)

- Single horizontal bar animating from 0.85 to post-investigation value
- Width proportional to score (0–1 scale), height 16px
- Color interpolation: green (`#3fb950`) at 0.8+, yellow (`#e3b341`) at 0.4–0.8, red (`#f85149`) below 0.4
- Transition: CSS `transition-all duration-500 ease-out` on width change
- Value label: right-aligned `text-[11px] font-mono text-text-main`
- Panel label: "Trust Score" 11px Inter 400 uppercase `text-text-muted`

**Source:** DASH-06, CONTEXT.md §Claude's Discretion (animation timing)

### Decision Log (DASH-08)

- Scrollable log, max-height ~120px, `overflow-y-auto`
- Each entry: timestamp (Code/Mono role — 12px Roboto Mono `text-text-muted`) + one-line attribution (12px Inter `text-text-main`)
- New entries prepend with `card-enter` slide-in animation (existing keyframe in index.html)
- Gate decision verdict badge inline: same color encoding as gate decision panel

**Source:** CONTEXT.md D-03 (implied below gate decision), DASH-08

### Aerospike Latency Display (DASH-09)

- Small metric chip at the bottom of the right column
- Format: `Aerospike: {latency}ms` — Code/Mono role: Roboto Mono 12px `text-text-muted`
- Value updates live from WebSocket `episode_written` event data
- Target indicator: green dot if <5ms, yellow if 5–20ms, red if >20ms

**Source:** CONTEXT.md D-03 (position — bottom of right column), DASH-09, MEM-05

## WebSocket Event → UI State Mapping

| Event | Zustand Action | Visual Effect |
|-------|----------------|---------------|
| `investigation_started` | `investigationStatus = 'running'`, reset nodes/edges | Disable attack buttons, supervisor node → active |
| `agent_completed` (risk) | `agents.risk.status = 'complete'`, store verdict | Risk node → complete |
| `agent_completed` (compliance) | `agents.compliance.status = 'complete'`, store verdict | Compliance node → complete |
| `agent_completed` (forensics) | `agents.forensics.status = 'complete'`, store verdict | Forensics node → complete |
| `verdict_board_assembled` | `verdictBoard = payload` | Populate verdict board table |
| `gate_evaluated` | `gateDecision = payload`, `investigationStatus = 'complete'` | Show gate decision, enable attack buttons, animate score bar, animate trust score |
| `episode_written` | `aerospikeLatencyMs = payload.latency_ms` | Update latency chip |
| `rule_generating` | append to `streamingBuffer`, `ruleStreaming = true` | Append token to rule source panel |
| `rule_deployed` | `ruleSources.push(payload)`, `ruleStreaming = false`, add rule node | Replace streamed content with final source, new rule node in tree |

**Source:** CONTEXT.md D-13, D-14, D-15, `sentinel/schemas/events.py` (all 9 EventType values)
