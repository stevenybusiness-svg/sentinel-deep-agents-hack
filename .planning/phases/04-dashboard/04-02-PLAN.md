**1. Create `frontend/src/components/GateDecisionPanel.jsx`:**

Panel showing gate decision with confirm button. Consumes `gateDecision` and `currentEpisodeId` from store. Also displays trust score inline (DASH-06 content folded into this panel per D-03 — no separate TrustScoreBar component).

Layout (per UI-SPEC gate decision panel):
```
+-----------------------------------------------------+
|  [icon] NO-GO                          Score: 1.84  |
|  Attribution: rule_mismatch, generated_rule_001 ... |
|  Trust: [====--------] 0.32                          |
|                                                     |
|  [ Confirm Attack -- Learn > ]                       |
+-----------------------------------------------------+
```

**Gate verdict text:** 20px Inter 600 (use `text-xl font-semibold`). Color: `text-danger` (NO-GO), `text-success` (GO), `text-warning` (ESCALATE).

**Score:** `text-[13px] text-text-muted font-mono` -- format: `Score: {value} > threshold`

**Attribution:** `text-[12px] text-text-main` -- wrap below score, shows `gateDecision.attribution`

**Trust score inline (DASH-06):**
Below attribution, add a small trust score bar:
```jsx
<div className="flex items-center gap-2 mt-2">
  <span className="text-[11px] uppercase text-text-muted">Trust</span>
  <div className="flex-1 h-3 bg-bg-dark rounded overflow-hidden">
    <div
      className="h-full rounded transition-all duration-500 ease-out"
      style={{
        width: `${trustScore * 100}%`,
        backgroundColor: trustScore >= 0.8 ? '#3fb950' : trustScore >= 0.4 ? '#e3b341' : '#f85149'
      }}
    />
  </div>
  <span className="text-[11px] font-mono text-text-main w-8 text-right">
    {trustScore.toFixed(2)}
  </span>
</div>
```
Consumes `trustScore` from store. Animates from initial 0.85 to post-investigation value via CSS transition. Color: green >= 0.8, yellow 0.4-0.8, red < 0.4.

**Confirm button (per D-06, D-07):**
- Only visible when `gateDecision?.decision === 'NO-GO'`
- Text: "Confirm Attack -- Learn >"
- Style: `bg-accent text-white font-semibold text-sm px-4 py-2 rounded hover:opacity-90 transition-opacity`
- On click: call `POST /api/confirm` with `{ episode_id: currentEpisodeId, attack_type: 'confirmed' }`
- After click: button text changes to "Generating rule..." and becomes disabled (use local state `confirming`)
- Reset `confirming` to false when a new `investigation_started` event resets the store

**Empty state:** When `gateDecision` is null, show: "No verdict yet." `text-text-muted text-[13px]`

**Panel wrapper:** `bg-surface rounded-lg border border-border-muted p-4`

**2. Create `frontend/src/components/AnomalyScoreBar.jsx`:**

Horizontal segmented bar showing each rule's contribution to the composite score.

Consumes `gateDecision.rule_contributions` and `gateDecision.composite_score` from store.

**Bar layout (per UI-SPEC):**
- Container: full panel width, height 24px (`h-6`)
- Background: `bg-bg-dark rounded overflow-hidden`
- Each rule contribution is a color-coded segment:
  - Hardcoded rules (`is_generated === false`): `bg-primary` (#57abff)
  - Generated rules (`is_generated === true`): `bg-danger` (#f85149)
  - Width proportional to score contribution relative to max(composite_score, 1.0): `width: ${(contribution.score / Math.max(composite, 1.0)) * 100}%`
- Segments rendered as inline `<div>` elements inside a flex container
- Threshold line: 2px wide vertical line at the 1.0 position: `left: ${(1.0 / Math.max(composite, 1.0)) * 100}%`, color `bg-accent`, absolute positioned

**Score label:** Right-aligned `text-[12px] font-mono text-text-main` showing composite score value

**Section label:** "ANOMALY SCORE" -- `text-[11px] font-semibold uppercase tracking-wider text-text-muted mb-1`

**Rule contribution labels:** Below the bar, show each rule_id and score in `text-[10px] font-mono text-text-muted`

**Empty state:** Show empty bar with threshold line only when no gate decision.
